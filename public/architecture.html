Smart Alumni Connect

System Architecture & Job Referral Platform

Comprehensive architectural documentation for a scalable, serverless platform connecting students and alumni through intelligent NLP matching and automated workflows.

Architectural Overview

1. Frontend Layer

  • ▹ Framework: React.js
  • ▹ Styling: Tailwind CSS
  • ▹ Hosting: Vercel (Edge Deployed)
  • ▹ Student Portal: Dashboard, Upload, Jobs, Mentorship, Readiness Score, Notifications
  • ▹ Alumni Portal: Dashboard, Requests, ACKs, Post Jobs, Notifications

2. Backend Layer

  • ▹ Architecture: Serverless
  • ▹ Runtime: Node.js & Express.js
  • ▹ Compute: AWS Lambda
  • ▹ Gateway: Amazon API Gateway

3. Database & Storage

  • ▹ Document DB: MongoDB (Profiles, Jobs)
  • ▹ Transactional: AWS DynamoDB / RDS
  • ▹ Object Storage: AWS S3 (Resumes, Avatars)

4. Core Integrations

  • ▹ ITSM: ServiceNow (Workflows)
  • ▹ ML Engine: NLP Resume Parsing & Scoring
  • ▹ Matching: Readiness Score to Job Postings

User Workflows (Data Flow)

1
Student Action

Uploads resume (to S3) and requests mentorship or a job referral.

2
ML Processing

System parses resume data via NLP, calculates "Placement Readiness Score", and maps recommended jobs.

3
Alumni Review

Receives requests in the Alumni Portal and triggers "Mentorship ACK" or "Referral ACK".

4
ServiceNow Sync

Referral pipelines to ServiceNow for automated tracking, ticketing, and real-time status updates.

API Design Considerations

RESTful Endpoints

Resource-oriented routing mapping cleanly to entities (e.g., /api/v1/students, /api/v1/jobs).

Stateless Authentication

JWT (JSON Web Tokens) used exclusively, validated securely at the AWS API Gateway layer via Custom Authorizers.

Pagination & Filtering

Cursor or offset-based pagination essential for Job Boards to minimize DynamoDB/MongoDB read units and improve response times.

Asynchronous Processing

Heavy operations (ML Parsing, ServiceNow syncing) return 202 Accepted and process via AWS SQS/SNS to prevent blocking.

Idempotency

Critical for state-changing operations like ACK/NACK to ensure network retries don't duplicate ServiceNow tickets.

Multipart Uploads

Bypassing heavy backend processing by using Pre-signed S3 URLs generated by API Gateway for secure, direct-to-bucket uploads.
